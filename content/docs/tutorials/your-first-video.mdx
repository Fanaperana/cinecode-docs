Now that the engine works, let's make something of your own. This lesson is not
one big copy-paste. You'll build the scene in small render checkpoints:

1. Start with a title.
2. Give the title a stronger visual style.
3. Add real source code.
4. Make the code type, spotlight, and move under the camera.
5. Change the source and render again.

That loop is the whole CineCode habit: **change one thing, render, watch what it
did**.

## Step 1 — Scaffold a project

The `new` command creates a ready-to-render project:

```bash
./target/release/cinecode new my-video
```

That gives you exactly three files — and **those three files are the whole idea**
behind CineCode:

```text
my-video/
├── codescene.toml      # ① the manifest — settings & which scenes to play
├── scenes/
│   └── intro.ccs       # ② the director's script — how to film the code
└── src/
    └── main.rs         # ③ your code — the thing being filmed
```

<Callout type="info">
  Think of it like a film set: `src/main.rs` is the **actor** (your real code),
  `intro.ccs` is the **director** (telling the camera what to do), and the
  rendered `.mp4` is the **finished film**. The script never *contains* your
  code — it `read(...)`s it.
</Callout>

## Step 2 — Meet the three files

Open the project and look at each file. The most important thing to notice: the
script doesn't hold the code, it **reads** `src/main.rs` and then says *how* to
animate it.

<Tabs items={['③ src/main.rs — your code', '② scenes/intro.ccs — the script', '① codescene.toml — settings']}>

<Tab value="③ src/main.rs — your code">

This is ordinary Rust — the thing you actually want to show. CineCode never
edits it; it only films it. Change this file and the video changes on the next
render.

```rust title="src/main.rs"
fn main() {
    let greeting = "Hello, CineCode!";
    println!("{greeting}");
}
```

</Tab>

<Tab value="② scenes/intro.ccs — the script">

The script is the storyboard. `read("src/main.rs")` pulls in the code from the
other tab, and the chained calls say *how* to animate it.

```rust title="scenes/intro.ccs"
// scene(name, duration_in_frames) — at 30 fps, 150 frames = 5 seconds
let s = scene("Intro", 150);

// A title that fades in, with a soft glow
s.text("Hello, CineCode", 96, 48).font_size(52).glow(0.5).fade_in(0, 18);

// Load src/main.rs and type it out like a keyboard
let code = s.code(read("src/main.rs"), "rust", 96, 150);
code.font_size(34).shadow(0.5).fade_in(0, 16).typewriter(6, 70);
code.spotlight(2, 0.6, 90, 18);   // at frame 90, brighten line 2, dim the rest

// A documentary camera that pushes in on line 2, then pulls back
let cam = s.camera();
cam.focus_on(code, 96, 24, 1.6, 2);
cam.zoom_to(1, 130, 24);
```

What each line does:

- `scene("Intro", 150)` — a 150-frame (5-second) scene.
- `s.text(...)` — a title at pixel `(96, 48)`, faded in over frames 0–18.
- `s.code(read("src/main.rs"), "rust", ...)` — loads and syntax-highlights your
  real code.
- `.typewriter(6, 70)` — types it out, starting at frame 6, over 70 frames.
- `.spotlight(2, 0.6, 90, 18)` — at frame 90, brightens line 2 and dims the rest.

</Tab>

<Tab value="① codescene.toml — settings">

The manifest says which scenes to play, the visual theme, and the output size.

```toml title="codescene.toml"
scenes = ["scenes/intro.ccs"]
theme = "dark_documentary"

[episode]
title = "my-video"

[output]
fps = 30
width = 1280
height = 720
```

<Callout type="warn">
  In TOML, all bare keys (`scenes`, `theme`, `fonts`) must appear **before** any
  `[table]`. If you see a parse error, check that ordering first.
</Callout>

</Tab>

</Tabs>

## Step 3 — Start with a title

Open `scenes/intro.ccs` and temporarily reduce it to only a title:

```rust title="scenes/intro.ccs"
let s = scene("Intro", 120);

s.text("Hello, CineCode", 96, 80)
    .font_size(56)
    .fade_in(0, 18);
```

Render it:

```bash
./target/release/cinecode render my-video -o title.mp4
```

<Video src="first-video-title" caption="Checkpoint 1: a single title fading in. Expand the source bundle to see the tiny scene that made it." />

What you just learned:

- `scene("Intro", 120)` creates a 120-frame scene. At 30 fps, that is 4 seconds.
- `s.text(...)` places text at `(x, y)` pixels from the top-left.
- `.font_size(56)` makes it large enough to read.
- `.fade_in(0, 18)` fades it in from frame 0 to frame 18.

This is the first useful CineCode unit: **create a node, then chain animation
methods onto it**.

## Step 4 — Style the title

Now make the title feel more intentional. Replace the title block with this:

```rust title="scenes/intro.ccs"
let s = scene("Intro", 120);

s.text("Hello, CineCode", 96, 80)
    .font("Georgia")
    .font_size(64)
    .glow(0.5)
    .slide_in("up", 50, 0, 24);
```

Render again:

```bash
./target/release/cinecode render my-video -o title-styled.mp4
```

<Video src="first-video-title-styled" caption="Checkpoint 2: the same title with font, glow, size, and motion added." />

What changed:

- `.font("Georgia")` asks for a system font by name.
- `.glow(0.5)` adds a soft documentary-style glow.
- `.slide_in("up", 50, 0, 24)` starts the title 50 pixels lower and slides it
  upward over 24 frames.

## Step 5 — Add code to the frame

Put the starter scene back together, but read it from the bottom up: title first,
then code.

```rust title="scenes/intro.ccs"
let s = scene("Intro", 150);

s.text("Hello, CineCode", 96, 48)
    .font_size(52)
    .glow(0.5)
    .fade_in(0, 18);

let code = s.code(read("src/main.rs"), "rust", 96, 150);
code.font_size(34)
    .shadow(0.5)
    .fade_in(0, 16)
    .typewriter(6, 70);
```

Render this checkpoint:

```bash
./target/release/cinecode render my-video -o code.mp4
```

<Video src="first-video-code" caption="Checkpoint 3: title plus real Rust source code read from src/main.rs." />

Now there are two different node types in one scene:

- `s.text(...)` creates normal title text.
- `s.code(read("src/main.rs"), "rust", ...)` reads a real file, syntax
  highlights it as Rust, and places it on screen.
- `.typewriter(6, 70)` reveals the code like typing, starting at frame 6.

The clip above is the first point where the scene starts to feel like a code
documentary: the page title gives context, while the code block gives the thing
being explained.

## Step 6 — Make the code explain itself

Code on screen is useful, but documentary code needs direction. Add a spotlight
after the typewriter:

```rust title="scenes/intro.ccs"
code.spotlight(2, 0.6, 90, 18);
```

That means: at frame 90, brighten line 2 and dim the other lines by `0.6` over
18 frames.

Then add a camera move:

```rust title="scenes/intro.ccs"
let cam = s.camera();
cam.focus_on(code, 96, 24, 1.6, 2);
cam.zoom_to(1, 130, 24);
```

Render again:

```bash
./target/release/cinecode render my-video -o spotlight.mp4
```

What you can do with this pattern:

- Spotlight the line your narration is talking about.
- Underline or point at a line when a small detail matters.
- Push the camera into the code, then pull back for context.
- Repeat spotlight calls to walk the viewer through an algorithm.

This example shows the next version of that idea: the spotlight walks from one
line to another while the code stays readable.

One command turns the three files into a video:

```bash
./target/release/cinecode render my-video -o my-video.mp4
```

## Step 8 — Change something and re-render

Make the loop click. Open `src/main.rs` and change just the greeting:

```rust title="src/main.rs"
fn main() {
    let greeting = "Rivers carve canyons.";
    println!("{greeting}");
}
```

Render again — you don't touch the script at all, because the script just
*reads* the file:

```bash
./target/release/cinecode render my-video -o my-video.mp4
```

The new text types itself out. That's the entire workflow: **edit your code (or
the script), render, watch.**

Before moving on, try one small change from each category:

- Change the title text in `scenes/intro.ccs`.
- Change `.font_size(52)` to `.font_size(44)`.
- Change `.glow(0.5)` to `.glow(0.9)`.
- Change `code.spotlight(2, ...)` to `code.spotlight(3, ...)`.
- Change the greeting in `src/main.rs`.

Render after each one. You will learn faster from five tiny renders than from
one large script you do not yet understand.

spotlighting and dimming lines like a real documentary.
