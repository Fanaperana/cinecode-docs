A documentary isn't just text on a screen — it *guides attention*. In this
lesson you'll make a block of code perform: lines light up, others dim, and the
code even edits itself.

We'll keep using the `my-video` project from the
[previous lesson](/docs/tutorials/your-first-video).

## Step 0 — Give yourself some code to walk through

The starter `src/main.rs` is only three lines — not much to spotlight. Replace it
with a small function so there are real lines to direct attention to:

```rust title="src/main.rs"
fn divide(a: i32, b: i32) -> i32 {
    let result = a / b;
    println!("{a} / {b} = {result}");
    return result;
}
```

Everything below animates **this** file. Keep it open — when the script says
“line 2,” it means line 2 *here* (`let result = a / b;`).

## Step 1 — Spotlight a line

A spotlight raises one line to full brightness and dims the rest. Add this after
your `typewriter` call:

```rust title="scenes/intro.ccs"
let code = s.code(read("src/main.rs"), "rust", 96, 150);
code.font_size(38).fade_in(0, 16).typewriter(6, 70);

// After typing finishes (~frame 90), spotlight line 2 — the division
code.spotlight(2, 0.6, 90, 18);
```

`spotlight(line, dim, start, dur)`:

- `line` — the 1-based line to highlight
- `dim` — how much to fade the others (0 = untouched, 1 = invisible)
- `start`, `dur` — when the spotlight ramps in, in frames

<Callout type="info">
  Lines are **1-based**, matching your editor. Line 1 is the first line of the
  block you passed to `s.code()`.
</Callout>

## Step 2 — Move the spotlight

Call `spotlight` again to shift focus to another line. Each call animates from
the previous state:

```rust
code.spotlight(2, 0.6, 92, 18);   // focus line 2 — the division
s.underline(code, 2, 98, 12);     // underline it as you explain
code.spotlight(3, 0.6, 150, 24);  // then slide focus to line 3 — the println
code.unfocus(200, 16);            // release everyone at the end
```

This is the core grammar of a CineCode walkthrough: **type the code, then walk
the spotlight down it** as your narration explains each part. Here is that exact
script, rendered against the `divide` function above:

## Step 3 — Underline and point

Spotlights aren't the only way to draw the eye. You can underline a line or
drop an arrow that points at it:

```rust
// Animate an underline beneath line 3
s.underline(code, 3, 100, 20);

// An arrow that points at line 4 from a screen position
s.point_at(code, 4, 700, 320, 120, 20);
```

See [Highlight & focus code](/docs/how-to/highlight-and-focus-code) for the full
set, including `surround` (a box around a range of lines).

## Step 4 — Edit the code live

Code can change on screen — perfect for showing a refactor. The example project
ships this real file, where `max` could be `0` and crash the divide:

```rust title="src/edit_source.rs (in the example project)"
fn normalize(values: &mut [f32]) {
    let max = peak(values);
    for v in values.iter_mut() {
        *v = *v / max;
    }
}
```

The script reads that file, spotlights the risky `let max` line, then **inserts
a guard clause** so the function is safe — all on screen. The source file itself
is never touched; the *animation* does the editing:

<Video src="edit_source" caption="A live refactor of src/edit_source.rs: spotlight the risky line, then insert a guard so normalize() is safe." />

Expand **Files that made this video** here and compare `src/edit_source.rs` with
`scenes/edit_source.ccs`. The Rust file is the before-state; the scene script is
where the new guard clause is animated into place.

For a single line, the simplest pattern is `backspace_to` and `retype` (covered
in detail in [Edit code live](/docs/how-to/edit-code-live)):

```rust
let code = s.code(read("src/main.rs"), "rust", 96, 160);
code.fade_in(0, 12).typewriter(6, 60);

// Remove back to a kept prefix, then type the corrected version
code.backspace_to("    return a / b", 90, 20);
code.retype("    if b == 0 { return 0 }\n    return a / b", 110, 40);
```
