This page lists the builders available on a scene. Animation, style, and focus
methods that you chain onto the returned nodes are documented in
[Code & node animations](/docs/reference/code-animations).

## `scene(name, frames)`

Declare a scene and its duration. Use raw frames for precision, or time helpers
for readability.

```rust
let s = scene("Introduction", sec(6));
let quick = scene("Title card", ms(2500));
```

`ms(n)` converts milliseconds to frames. `sec(n)`, `secs(n)`, `seconds(n)`, and
`s(n)` convert seconds to frames using `[output].fps` from `codescene.toml`.

## Timeline markers

Use markers when a scene has more than a couple of timed actions. Markers keep
the script readable because animation calls can refer to named moments instead
of repeating raw offsets.

```rust
mark("code_in", sec(1));
mark("focus_line", after("code_in", sec(1.4)));

code.fade_in(at("code_in"), ms(500));
code.typewriter(after("code_in", ms(200)), sec(2.2));
code.spotlight(2, 0.6, at("focus_line"), ms(650));
```

| Function | Description |
| --- | --- |
| `mark(name, time)` | Store a named timeline position and return its frame. |
| `at(name)` | Read a named timeline position. |
| `after(name, offset)` | Return a marker plus an offset. |
| `after(time, offset)` | Return one time value plus another. |

## Text

```rust
// text(content, x, y)
let title = s.text("Hello, CineCode", 96, 64);
```

## Code

```rust
// code(source, lang, [theme], x, y)
let a = s.code(read("src/main.rs"), "rust", 96, 160);
let b = s.code(read("src/main.rs"), "rust", "Solarized (dark)", 96, 160);
```

- `source` — the code text (use `read("path")` to load a file)
- `lang` — a language id (see [Languages](/docs/reference/languages))
- `theme` — optional code highlight theme (see [Themes](/docs/reference/themes))
- `x, y` — top-left position

<Callout type="info">
  `read("src/main.rs")` returns the **literal contents** of that file as a
  string — the script never rewrites your code, it only loads and films it. You
  can also pass an inline string, e.g. `s.code("let x = 1;", "rust", 96, 160)`.
  See [How it works — three ingredients](/docs) for the full picture.
</Callout>

## Shapes

```rust
s.panel(x, y, w, h, r, g, b);
s.rect(x, y, w, h);
s.square(x, y, size);
s.circle(x, y, radius);
s.ellipse(x, y, rx, ry);
s.line(x1, y1, x2, y2);
s.polygon([[x, y], ...]);

```

## Images

```rust
// image(path, x, y, [w, h])
s.image("assets/diagram.png", 700, 160, 420, 300);
```

## Focus helpers

These act on a code/text node:

```rust
// underline(node, line, start, dur)
s.underline(code, 3, sec(3), ms(667));

// surround(node, from_line, to_line, start, dur)
s.surround(code, 1, 4, sec(3), ms(800));

// point_at(node, line, from_x, from_y, start, dur)
s.point_at(code, 5, 760, 320, sec(4), ms(600));

// variable_update(node, line, label, start, dur) — a value badge beside a line
s.variable_update(code, 4, "acc = 6", sec(4), ms(250));

// diff_add / diff_remove (node, line, start, dur) — mark one changed line
s.diff_add(code, 3, sec(4), ms(450));
s.diff_remove(code, 2, sec(4), ms(450));

// diff(node, added_lines, removed_lines, start, dur) — a whole patch at once
s.diff(code, [3], [2], sec(4), ms(450));
```

## Morph

```rust
// morph(old_node, new_node, style, start, dur)
s.morph(a, b, "dissolve", 60, 40);
```

## Link code to a simulation

```rust
// link(node, line, sim, start, dur)
s.link(code, 9, grid, 100, 90);
```

See [Bind code to simulations](/docs/how-to/bind-code-to-simulations).

## Simulations

```rust
s.terrain(x, y);    s.noise(x, y);
s.pathfind(x, y);
s.particles(from_x, from_y, to_x, to_y);
s.graph(x, y);
s.life(x, y);       s.automata(x, y);
s.boids(x, y);
```

Each returns a simulation node configured with a fluent chain — see
[Simulations](/docs/reference/simulations).

let cam = s.camera();

// transition(kind, frames, [r, g, b])
